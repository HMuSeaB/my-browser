# MyBrowser 维护冒烟清单

这份清单用于日常维护、功能回归和发版前自查，默认基线为当前桌面版。

## 默认校验命令

按这个顺序执行：

1. `go test ./...`
2. `cd frontend && npm run build`
3. `wails build -clean -f`

默认 Windows 构建产物：

- `build/bin/my-browser.exe`

## 环境管理

1. 新建环境，验证可留空分类和默认页。
2. 新建环境时填写分类，确认首页筛选栏会出现对应分类。
3. 编辑环境，确认分类、代理、默认页保存后重新打开仍一致。
4. 启动环境，关闭浏览器后手动执行同步，确认状态未报错。
5. 环境详情页中的 `启动环境 / 自动化打开 / 管理环境` 都能正常进入对应流程。

## 代理与导入导出

1. 新增一个代理并执行测试。
2. 删除刚创建的代理，确认代理池状态刷新正常。
3. 导出环境包，再导入环境包，确认名称冲突时会自动重命名。
4. 导入后的环境确认分类、默认页、代理、Cookie 文本仍存在或按默认值补齐。

## 自动化控制台

1. 启用自动化控制台，确认监听地址、Token、活动会话信息可见。
2. 对同一环境连续执行两次自动化打开，确认复用已有会话而不是重复创建。
3. 一次使用指定链接打开，一次使用环境默认页打开，确认两条路径都可用。
4. 停止自动化会话后，确认状态回落且不会阻止再次启动。

## 存储与桌面能力

1. 默认安装模式下确认数据目录位于 `%LOCALAPPDATA%\\MyBrowser`。
2. 放置 `portable.flag` 后确认切换到程序同目录 `MyBrowserData`。
3. 如存在旧版 `data` 目录，确认只在目标目录为空时才触发迁移。
4. 手动验证 `打开数据目录`、`注册默认浏览器`、`创建桌面快捷方式`。

## 界面回归观察点

重点留意这些高频交互：

- 环境分类筛选切换是否稳定
- 环境详情与首页卡片之间的状态联动
- 自动化控制台通知反馈是否清晰
- 浅色主题下按钮、状态标签、通知文字是否有足够对比度
- 长名称、长 URL、长分类是否会把卡片或详情页撑坏
